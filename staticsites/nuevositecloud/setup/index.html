Preparación

Intro

Este es un proceso paso a paso para crear un nuevo site basado en hugo sin depender del portátil. Para ello vamos a utilizar el cloudShell the GCP

Google Cloud Identity & Google Workspaces

Se puede utilizar GCP sin tener una organización pero en mi caso es preferible hacer un setup como una empresa lo haría (parcialmente porque tampoco soy un super experto)

  • Dominio adquirido y configurado por ejemplo en goDady
  • Una cuenta de pago con un solo usuario de Workspaces (unos 6 euros)
  • En Workspaces podemos añadir Cloud Identity para poder añadir hasta 50 usuarios sin tener que pagar licencia por cada uno de ellos.

La cuenta de Workspaces será utilizada para ser el organization admin de la cuenta de google cloud. Las cuentas de cloud identity se usarán para crear owners de proyectos, viewers y demas.

Perfiles de Chrome

Yo tengo un perfil de Chrome distinto para cada usuario. De este modo no tengo que estár constantmente entrando en modo incognito.

#### Permisos en GCP La cuanta de super admin de Workspaces podemos ver en Admin roles and privileges que es Super Admin. Ésto hace que por defecto sea Organization Administrator en GCP. Se pueden tener los siguientes roles

  • Folder Admin
  • Organisation Administrator
  • Organisation Policy Administrator
  • Project Creator

Habilitar multfactor authentication.

#### Cuenta de Developer Ahora se puede crear una cuenta dev@dominioquesea.com en Workspaces pero en Cloud Identity Free para que no consuma licencia. Con la cuenta de super admin en GCP añadimos ese usuario a GCP

Permisos del develoer account

  • Owner
  • Browser

#### Creacion de Folder. Con la cuenta de super admin creamos una carpeta bajo la organización En IAM -> Manage Resources Create Folder

En esa misma pagina añadir a dev como admin de esa carpeta

Por ejemplo Creo una carpeta llamada Frontend Si esta MFA habilitado sera necesario habilitar en Users los codigos de backap para el primer login.

Proyect

Crear el proyecto con DEV para poder habilitar la facturacion podemos hacerlo con el super admin. Para eso superadmin ha de tener owner.
