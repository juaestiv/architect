Terraform Modules

Intro

The use of modules will allow to write less code and to reuse the resource creation in several environments. I am going to create a module which setup the following resources


  • VPC
  • Public Subnet
  • Private Subnet
  • Internet Gateway

Once the module is created it can be used to deploy Production and Staging environments without the need to repeat the code for those envirroments.

The idea is that it will be a variable that will replace the CIDR blocks acording the environment.

In other words, when creating Production it will use 10.1.0.0 network and when creating Staging it wil use 10.2.0.0.

Step 1. Create a folder tree

Create a folder tree as follows

yourproject
│   ├── modules
│   ├── prod
│   └── staging

Under modules directory I am going to create a reusable module for VPC and it’s subnets. For that purpose I’ll create a vpc_module folder that will contain main.tf and variables as follows.

% tree
.
├── modules
│   └── vpc_module
│       ├── main.tf
│       └── variables.tf
├── prod
└── staging

Step 2. Creating the VPC Module

Content of modules/vpc_module/variables.tf

Here we will define the network default address

variable "vpc_cidr" {
    type = string
    default = "10.1.0.0/16"
}

Explanation: This vpc_cidr variable will be replaced in the main.tf with different values depending of the environment for example

Content of modules/vpc_module/main.tf

Pay attention to the use of var.vpc_cidr. Another interesting topic is the use of the terraform function called cidrsubnet() Cidrsubnet calculates a subnet address within given IP network address prefix. It takes 3 parameters

  • the cidr block
  • newbits (if the network is /16 and we setup a number of 8 the it will create a /24 subnet)
  • num (subnet number)
resource "aws_vpc" "myvpc" {
    cidr_block = var.vpc_cidr
    enable_dns_hostnames = true
    enable_classiclink_dns_support = true
}

resource "aws_internet_gateway" "myigw" {
  vpc_id = aws_vpc.myvpc.id
}

resource "aws_subnet" "public" {
    cidr_block = cidrsubnet("${var.vpc_cidr}", 8, 1)
    vpc_id = aws_vpc.myvpc.id
}

resource "aws_subnet" "private" {
    cidr_block = cidrsubnet("${var.vpc_cidr}", 8, 2)
    vpc_id = aws_vpc.myvpc.id
}

Step 3. Prepare the prod and staging folders

Both folders will have the files

  • main.tf
  • variables.tf

So the whole tree will be

% tree
.
├── modules
│   └── vpc_module
│       ├── main.tf
│       └── variables.tf
├── prod
│   ├── main.tf
│   └── variables.tf
└── staging
    ├── main.tf
    └── variables.tf
Prod environment

The prod/variables.tf file will look like this

variable "prodcidr" {
    type = string
    default = "10.1.0.0/16"
}

Now when we invoke the module from prod/main.tf the cidr will use this variable see how below.

provider "aws" {
    region = "eu-central-1"
}
module "mymodule" {
    source = "../modules/vpc_module"
    vpc_cidr = var.prodcidr
    
}

On the code above we are calling the module (source = “../modules/vpc_module”) but we are using the var.prodcidr cidr

Staging environment

Follwing the same pattern I configure the staging

variable "stgcidr" {
    type = string
    default = "10.2.0.0/16"
}

provider "aws" {
    region = "eu-central-1"
}
module "mymodule" {
    source = "../modules/vpc_module"
    vpc_cidr = var.stgcidr
    
}

Step 4. Initialize prod and stading folders

Execute

terraform init

under

  • /prod
  • /staging

Execute

terraform plan

to see the proposed changes
